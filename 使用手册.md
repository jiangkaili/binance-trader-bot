## 1. 这个项目是什么

这是一个在币安（Binance）合约交易所自动交易的机器人。它的核心逻辑：

- 每 60 秒检查一次 BTC（比特币）的 5 分钟 K 线数据
- 当 RSI 指标出现极端值时（超卖做多、超买做空），自动开仓
- 开仓的同时在交易所端挂好止损（SL）和止盈（TP）
- 如果亏损达到日/周上限，自动停止开新仓
- 所有交易记录存入 SQLite 数据库，可随时查询

**重要提醒**：这是工程实验项目，不是赚钱机器。作者自己都在亏钱（起始 45.83 USDT，当前约 34 USDT）。加密货币合约杠杆交易可能导致本金全部损失。只用你能承受损失的钱来跑。

## 3. 第一步：安装 Python 环境

### Windows

1. 打开浏览器，访问 https://www.python.org/downloads/
2. 下载最新的 Python 3（选 "Windows installer (64-bit)"）

```bash

pip3 --version
```

### Linux (Ubuntu/Debian)

```bash
sudo apt update
```

## 6. 第四步：注册币安账号并获取 API Key

### 6.1 注册币安账号

1. 访问 https://www.binance.com
2. 注册账号并完成实名认证（KYC）
3. 在钱包里充入至少 20 USDT（先不要充太多）

### 6.2 开通合约账户

1. 登录币安后，顶部导航栏找到"合约"（Futures）→ "USDⓈ-M"
2. 首次进入会提示开通合约账户，按提示操作

### 6.3 创建 API Key

1. 访问 https://www.binance.com/en/my/settings/api-management
2. 点击"Create API"
3. 选择"System Generated"
4. 给 API 起个名字，比如"trading-bot"
5. 完成安全验证（邮箱 + 手机/Google Auth）
6. **权限设置**（非常重要！）：
   - ✅ 勾选 "Enable Futures"（启用合约交易）
   - ❌ 不勾 "Enable Withdrawals"（绝对不要给提现权限！）
   - 其他保持默认
7. **IP 白名单**（强烈建议）：
   - 勾选"Restrict access to trusted IPs only"
   - 填入你运行机器人的电脑/服务器的公网 IP
   - 查公网 IP：浏览器访问 https://api.ipify.org
8. 保存好两个值：
   - **API Key**：形如 `xBcD1234...`
   - **API Secret**：只显示一次，**务必保存**，丢了只能重建

### 6.4 获取测试网 Key（可选，但强烈建议先跑测试网）

测试网是币安提供的模拟交易环境，用假钱交易，行情是真实的。

1. 访问 https://testnet.binancefuture.com
2. 用你的币安账号登录
3. 页面右下角找到 "API Key" 区域
4. 点击 "Generate HMAC_SHA1 Key"
5. 保存生成的 API Key 和 Secret

## 7. 第五步：配置环境变量（密钥和网络）

项目有两个环境变量文件：
- `.env.testnet` — 测试网（默认使用这个）
- `.env` — 主网实盘

### 7.1 配置测试网（先跑测试网！）

编辑 `.env.testnet` 文件（用记事本、VS Code 或任意文本编辑器）：

```bash
# 文件路径：binance-trader-bot/.env.testnet

# 把下面两个值换成你在 6.4 步获取的测试网 Key
BINANCE_API_KEY=你的测试网API_Key
BINANCE_API_SECRET=你的测试网API_Secret

# 使用测试网
USE_TESTNET=true
```

### 7.2 配置主网（实盘时用）

复制模板并编辑：

```bash
# Windows PowerShell
cp .env.example .env
# Mac/Linux
cp .env.example .env
```

编辑 `.env` 文件：

```bash
# 文件路径：binance-trader-bot/.env

# 你在 6.3 步获取的主网 API Key
BINANCE_API_KEY=你的主网API_Key
BINANCE_API_SECRET=你的主网API_Secret

# 主网（关闭测试网）
USE_TESTNET=false
```

### 7.3 配置代理（仅中国大陆用户需要）

如果你在中国大陆，币安 API 被墙，需要配置代理。在 `.env` 或 `.env.testnet` 中设置：

```bash
# 假设你的 VPN/代理跑在本地 12000 端口
PROXY_HOST=127.0.0.1
PROXY_PORT=12000
```

如果你不确定代理端口，先启动你的 VPN 软件，查看它的本地代理端口（通常是 1080、7890、12000 等）。

**不需要代理的地区**（海外、港澳等）：把 PROXY_HOST 留空、PROXY_PORT 设为 0 即可。

### 7.4 安全检查

确认 `.gitignore` 文件中包含以下行（防止密钥被提交到 GitHub）：

```
.env
.env.*
!.env.example
```

项目默认已配置好，但检查一下不亏。**如果你的 API Key 不小心被提交到了 GitHub，立刻去币安删除这个 Key 重建一个！**

## 8. 第六步：配置策略参数

打开 `config/trader.yaml`，这是所有策略和风控参数的配置文件。以下是当前 v7 版本的配置，每个参数都有注释说明：

```yaml
# ===== 交易对 =====
symbol: BTCUSDT               # 交易对，目前只支持 BTCUSDT

# ===== 仓位大小 =====
target_position_usdt: 15.0    # 每次开仓使用的保证金（USDT）
                               # 15 USDT × 5x杠杆 = 75 USDT 名义价值
                               # 建议不超过账户余额的 50%

# ===== 杠杆 =====
leverage: 5                    # 杠杆倍数。5x = 价格波动 1% → 保证金变动 5%
                               # 新手不要超过 5x！

# ===== 策略：RSI 均值回归 =====
strategy_name: rsi_extremes_5m
rsi_period: 7                  # RSI 计算周期（7根5分钟K线）
rsi_oversold: 15.0             # RSI < 15 时做多（超卖反弹）
rsi_overbought: 85.0           # RSI > 85 时做空（超买回落）

# ===== K线/轮询 =====
kline_interval: 5m             # K线周期：5分钟
poll_seconds: 60               # 每60秒检查一次
warmup_bars: 210               # 启动后至少需要210根K线才开始交易
                               # 210根 × 5分钟 = 17.5小时预热期

# ===== 止损/止盈 =====
stop_loss_pct: 0.015           # 止损：1.5%价格变动
                               # 5x杠杆下 = 7.5%保证金损失 ≈ -1.13 USDT
take_profit_pct: 0.030         # 止盈：3.0%价格变动
                               # 5x杠杆下 = 15%保证金收益 ≈ +2.25 USDT
                               # 盈亏比 = 1:2

# ===== 信号退出 =====
disable_signal_exit: true      # true = 只靠SL/TP平仓，不靠反向信号
                               # 防止赢家被过早平仓

# ===== 风控熔断 =====
daily_loss_pct: 0.25           # 当日亏损达25% → 停止开新仓
weekly_loss_pct: 0.40          # 本周亏损达40% → 停止开新仓

# ===== 连亏保护 =====
streak_cooldown_hours: 24      # 连亏3次后冷却24小时
streak_loss_count: 3

# ===== ADX 趋势过滤 =====
trend_filter_enabled: true
trend_filter_adx_threshold: 25.0  # ADX > 25 = 强趋势，不开仓
                                   # 均值回归在趋势中会亏钱

# ===== EMA 趋势对齐过滤（v7新增）=====
trend_ema_filter_enabled: true
trend_ema_period: 200           # 只在价格 > EMA200 时做多
                                # 只在价格 < EMA200 时做空
                                # 从根源避免逆势交易

# ===== 交易冷却 =====
cooldown_bars_after_trade: 12   # 平仓后等待12根K线（1小时）再开新仓
```

### 新手建议参数

如果你是新手，建议先用更保守的参数：

```yaml
target_position_usdt: 5.0      # 更小的仓位
leverage: 3                     # 更低的杠杆
stop_loss_pct: 0.02            # 更宽的止损（给价格更多波动空间）
take_profit_pct: 0.04           # 更高的止盈
daily_loss_pct: 0.15            # 更严格的日亏损上限
```

## 9. 第七步：用测试网跑通流程（不花真钱）

测试网用假钱，行情是真实的，适合验证机器人是否正常工作。

### 9.1 启动

```bash
# 确保在项目目录下，且虚拟环境已激活
# 确认 .env.testnet 已填入测试网密钥

# 测试网 dry-run（只看信号，连假单都不下）
python scripts/live_trader.py --dry-run --env-file .env.testnet

# 测试网实盘（用假钱下真单）
python scripts/live_trader.py --env-file .env.testnet
```

### 9.2 预期输出

启动后你会看到类似这样的输出：

```
Mode : DRY-RUN
Base : https://testnet.binancefuture.com
Env  : .env.testnet
Key  : ...xxxx  (redacted)

============================================================
LIVE MODE — REAL MONEY AT RISK

  - Daily cap  -25% of starting equity
  - Weekly cap -40% of starting equity
  - Single position, BTCUSDT only, 15.0 USDT, 5x leverage
  - To stop gracefully:  kill -TERM <pid>

[2026-06-28T00:00:00+00:00] [INFO] STARTED symbol=BTCUSDT leverage=5x ...
[2026-06-28T00:10:00+00:00] [INFO] heartbeat tick=10 sig=FLAT pos=FLAT ...
```

### 9.3 验证要点

看到以下信息说明正常：
- `STARTED` 消息显示正确的杠杆和参数
- `heartbeat` 日志每 10 分钟出现一次（tick=10, 20, 30...）
- `sig=FLAT pos=FLAT` 表示当前无持仓无信号
- `adx=XX.X` 显示当前 ADX 值
- 没有 `[ERROR]` 消息

### 9.4 常见启动错误

| 错误 | 原因 | 解决方案 |
|------|------|---------|
| `BINANCE_API_KEY not set` | 密钥文件没填 | 检查 `.env.testnet` 文件内容 |
| `ProxyError` | 网络不通 | 检查代理设置，确保 VPN 已启动 |
| `Invalid API-key` | 密钥错误或权限不对 | 重新生成 API Key，确保勾选 Futures |
| `-1021 Timestamp` | 系统时间不准 | 同步系统时间（Windows 设置→时间→立即同步） |
| `ModuleNotFoundError` | 依赖没装好 | 重新 `pip install -r requirements.txt` |

按 `Ctrl+C` 停止机器人。

- ADX 过滤是否正常工作（`paused: trending` 消息）
- EMA 过滤是否正常工作
- 有没有报错

## 11. 第九步：实盘运行（真金白银）

> - ✅ 测试网跑通
> - ✅ Dry-run 跑了至少 24 小时无报错

## 12. 第十步：后台运行 + 自动重启（看门狗）

关掉终端机器人就停了。你需要让它后台运行，并在崩溃时自动重启。

### 方案 A：Linux / macOS / WSL（推荐）

#### 后台启动

```bash
# 用项目自带脚本
bash scripts/start_trader.sh
```

这会在后台启动机器人，PID 写入 `data/live_trader.pid`。

#### 设置 crontab 看门狗

```bash
# 编辑定时任务
crontab -e

# 添加这一行（每5分钟检查一次，如果机器人没在跑就重启）：
*/5 * * * * cd /你的路径/binance-trader-bot && bash scripts/watchdog.sh
```

#### 停止

```bash
bash scripts/stop_trader.sh
```

### 方案 B：Windows（PowerShell + Task Scheduler）

#### 创建启动脚本

在项目目录下创建 `start_bot.bat`：

```bat
@echo off
cd /d C:\你的路径\binance-trader-bot
call .venv\Scripts\activate.bat
pythonw scripts/live_trader.py --env-file .env
```

> 用 `pythonw` 而不是 `python`，这样不会弹出命令行窗口。

#### 设置任务计划程序（自动重启）

1. 打开"任务计划程序"（开始菜单搜索）
2. 创建基本任务 → 名称填 "Trader Watchdog"
3. 触发器：每日，每 5 分钟重复
4. 操作：启动程序
   - 程序：`powershell.exe`
   - 参数：
     ```
     -NoProfile -Command "if (!(Get-Process pythonw -ErrorAction SilentlyContinue)) { Start-Process -FilePath 'C:\你的路径\binance-trader-bot\start_bot.bat' -WindowStyle Hidden }
     ```
5. 完成

这样每隔 5 分钟检查一次，如果机器人没在跑就自动启动。

#### 手动停止

## 紧急停止：三种方法

### 方法 1：紧急开关（推荐，不关程序）

```bash
# 创建 KILLSWITCH 文件
touch data/KILLSWITCH      # Linux/Mac
New-Item data\KILLSWITCH   # Windows PowerShell
```

机器人检测到这个文件后，会停止开新仓，但**已有的持仓和止损止盈继续保留**。

取消紧急开关：
```bash
rm data/KILLSWITCH          # Linux/Mac
Remove-Item data\KILLSWITCH # Windows
```

### 方法 2：优雅停止（平仓后退出）

```bash
# Linux/Mac
bash scripts/stop_trader.sh
# 或
kill -TERM $(cat data/live_trader.pid)

# Windows
Stop-Process -Name pythonw -Force
```

机器人会尝试平仓后退出。但如果平仓失败，持仓会留在交易所，止损止盈仍然有效。

### 方法 3：交易所手动操作（最可靠）

1. 登录币安合约页面
2. 手动平仓
3. 手动撤销所有挂单
4. 然后停止机器人程序

**不管用哪种方法，都建议去币安 UI 确认没有遗留持仓和挂单。**

### 回测结果怎么看

回测输出的关键指标：
- `PnL` — 总盈亏（USDT）
- `N` — 交易笔数
- `WR%` — 胜率
- `TP` — 止盈触发次数
- `SL` — 止损触发次数

**重要**：回测结果不等于实盘表现。回测不含滑点、延迟、API失败等实盘损耗。一个策略在回测中正收益，实盘可能亏钱。

## 常见问题 FAQ

### Q1: 机器人启动后一直显示 "warmup"，不开仓？

这是正常的。机器人需要 `warmup_bars`（默认 210）根 K 线才能开始计算 EMA200 指标。210 根 × 5 分钟 = 约 17.5 小时。

如果你不想等，可以临时把 `warmup_bars` 改小，但 EMA200 会不准确。

### Q2: 一直显示 "paused: trending (ADX=XX > 25)"？

这说明市场处于强趋势中，ADX 过滤器阻止了开仓。这是设计行为——均值回归在趋势中会亏钱。等 ADX 降到 25 以下就会恢复。

### Q3: 一直显示 "paused: bearish/bullish (price >/< EMA200)"？

EMA 趋势过滤器在工作。比如价格在 EMA200 下方时，只允许做空不允许做多。如果 RSI 超卖（想做多）但价格在 EMA200 下方，机器人会跳过这个信号。

### Q4: "Timestamp for this request is outside the recvWindow"

你的系统时间不准。

- Windows：设置 → 时间和语言 → 立即同步
- Linux：`sudo ntpdate ntp.ubuntu.com`

### Q5: "ProxyError: Unable to connect to proxy"

你的代理没启动或端口不对。检查：
1. VPN 软件是否在运行
2. `.env` 中的 `PROXY_HOST` 和 `PROXY_PORT` 是否正确
3. 试着在浏览器里能不能打开 https://fapi.binance.com

### Q6: 止损止盈挂上了但看不到？

币安合约的算法订单（algoOrder）不在普通的"当前委托"列表里。要去"条件订单"或" algo Orders"页面看。或者用 `python scripts/list_algo_orders.py` 查。

### Q7: 机器人崩了，我的持仓会怎样？

**止损止盈还在**。因为 SL/TP 是直接挂在交易所端的（algoOrder），机器人崩溃不影响它们。但如果机器人在开仓和挂 SL/TP 之间崩溃，可能存在裸仓风险。建议用看门狗自动重启。

### Q8: 怎么改交易对？

编辑 `config/trader.yaml`，把 `symbol: BTCUSDT` 改成你想交易的，比如 `ETHUSDT`。但注意：
- 目前只测试过 BTCUSDT
- 不同币种的波动特性不同，参数需要重新回测
- 小币种可能有流动性问题

### Q9: 可以同时跑多个币种吗？

当前设计是单币种单仓位。要跑多个币种需要修改代码，不在本手册范围内。

### Q10: 机器人一直在亏钱怎么办？

1. 检查是不是在趋势行情中逆势交易（看 ADX 和 EMA 日志）
2. 考虑用 `touch data/KILLSWITCH` 暂停交易
3. 阅读项目根目录的 `策略归档.md`，了解之前版本的失败原因
4. 用回测工具测试新参数
5. 接受现实：这个项目是工程实验，不保证盈利

### Q11: 如何在 GitHub 上提出问题或建议？

在 https://github.com/jiangkaili/binance-trader-bot/issues 提交 Issue。请勿询问"保证赚钱的参数"或"投资建议"。
