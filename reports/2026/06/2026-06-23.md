### 4.2 核心问题：盈亏比严重失衡

**这是当前策略最致命的问题。** 数据显示：

- **盈利交易**：通过 RSI 信号平仓，平均盈利 +0.84 USDT（价格变动 0.03%~0.34%）
- **亏损交易**：通过交易所止损平仓，平均亏损 -4.12 USDT（价格变动 1%~1.2%）
- **盈亏比**：0.20，意味着每亏 1 USDT 只能赚回 0.20 USDT
- **盈亏平衡胜率**：需要 83% 胜率才能不亏钱（当前 67%）

**根因分析：** RSI 信号平仓在价格只移动 0.03%~0.34% 时就触发了（RSI 从 20 回到 80），而 1% 止损要承受价格移动 1% 的全额亏损。止盈（1%）几乎从未被触发，因为 RSI 信号总是先到。

**具体案例：**
- Trade #4（最近一笔）：入场 64000.1，RSI=19.3 → 平仓 64216.23，RSI=80.1，盈利 +1.73 USDT。但 1% 止盈在 64640.1，本可盈利 +5.12 USDT。信号平仓使利润缩水 66%。
- Trade #6：入场 64637.60，止损 63991.2，实际平仓 63890.8（滑点 -100 点），亏损 -5.97 USDT。

### 4.3 次要问题：止损滑点

Trade #6 的预期止损亏损为 -5.17 USDT（入场 64637.60 - 止损 63991.2）× 0.008，但实际亏损 -5.97 USDT，多亏 0.80 USDT（15.5%）。这是因为 STOP_MARKET 订单以市价执行，在快速下跌时有滑点。

## 五、代码修复

### 5.1 关键 Bug 修复：交易所侧止损/止盈未记录

**问题描述：** 当交易所的 STOP_MARKET 或 TAKE_PROFIT_MARKET 在两次轮询（60 秒间隔）之间触发并平仓时，bot 的下一次 `get_position()` 返回 None，但 bot 不会调用 `close_position()` 来记录这笔交易。导致：
- 已实现亏损未记入 `daily_pnl` / `weekly_pnl`
- 交易未记录到 SQLite 数据库
- 连亏检测（3 连败冷却）失效
- -10% 永久熔断检测失效（读取 `SUM(pnl) FROM trades`）
- **6.60 USDT 的亏损完全不可见**

**证据：**
- Binance 收入 API 显示 6 笔 REALIZED_PNL，但 trades 表只有 3 笔
- Jun 21 23:31 的 -2.272 USDT 亏损和 Jun 22 22:28 的 -5.9744 USDT 亏损均未入库
- bot 的 daily_pnl 显示 0.00，而实际当日已实现 -5.97 USDT 亏损

**修复内容（scripts/live_trader.py）：**

1. **新增 `_handle_external_close(prev_pos)` 方法**：当检测到持仓在两次轮询间消失（且非 bot 主动平仓）时，通过 `/fapi/v1/income` API 获取精确的已实现盈亏，记录到数据库和 daily/weekly pnl 计数器。

2. **新增 `_fetch_last_realized_pnl()` 方法**：查询 Binance 收入接口获取最近的 REALIZED_PNL。

3. **新增 `_check_streak()` 方法**：从 `close_position()` 中提取连亏检测逻辑，供 bot 平仓和外部平仓共用。

4. **修改 `tick()` 方法**：在获取持仓后，检测 `prev_position is not None and self.position is None` 的情况，调用 `_handle_external_close()`。

5. **修改 `close_position()` 方法**：使用提取出的 `_check_streak()` 替代内联代码。

## 七、建议

### 7.1 策略改进（优先级：高）

1. **调整盈亏比**：当前信号平仓利润（~0.84 USDT）远小于止损亏损（~4.12 USDT）。建议：
   - **方案 A**：收紧止损至 0.3%~0.5%，使其与典型信号利润匹配
   - **方案 B**：移除信号平仓，仅依赖 1% 止损/止盈
   - **方案 C**：使用移动止盈（trailing stop）替代固定止盈，让利润奔跑
   - **推荐**：先回测方案 A（止损 0.3%，止盈 1%），保持信号平仓逻辑不变

2. **增加信号平仓的缓冲区**：当前 RSI 从 20 回到 80 就平仓，可改为 RSI 回到 50（中性区）才平仓，让趋势发展更充分。但这需要回测验证。

3. **RSI 参数**：7 周期 RSI 在 5m K线上较为敏感，可考虑 14 周期 RSI 配合 25/75 阈值，减少假信号。

### 7.2 风控改进（优先级：中）

1. **止损滑点**：考虑使用 LIMIT 止损单（在触发价附近限价）替代 STOP_MARKET，减少滑点。但需承担极端行情下不成交的风险。
2. **下单失败冷却**：保证金不足或下单失败后，应增加 5~10 分钟冷却期，避免每分钟重复尝试。
3. **重入场冷却**：止损触发后，应增加 15~30 分钟冷却期再允许开新仓，避免在趋势中连续止损。

### 7.3 需要关注的风险点

1. **累计亏损已达 -10.7%**（相对 45.83 起始权益），熔断即将触发
2. **INTCUSDT 手动仓位**占用 5.77 USDT 保证金，减少了 bot 的可用余额（仅剩 8.15 USDT）。如果两个仓位同时亏损，可能面临保证金不足
3. **bot 的 daily_pnl / weekly_pnl 仍然不准**（未包含回填的历史亏损）。代码修复后，未来的交易所侧平仓将被正确记录，但历史数据已无法追溯修正内部计数器
4. **频繁的配置变更**使策略评估困难，建议固定一组参数运行至少 1~2 周再评估
